# 修复与优化计划

检测到 `npm run dev` 异常退出，且代码在缺少环境变量时会直接抛出致命错误。为了修复此问题并提升开发体验，我将执行以下操作：

## 1. 增强代码容错性 (Mock 模式)
修改 `lib/notion.ts`，实现“开发模式”回退机制：
- 当检测到 `NOTION_API_KEY` 或 `NOTION_DATABASE_ID` 缺失时，不再直接抛出 Crash 错误。
- 改为返回**模拟数据 (Mock Data)**（例如一篇 "Welcome to Notion Blog" 的示例文章）。
- 在控制台输出配置缺失的警告，指导用户进行配置。

## 2. 恢复服务
- 清理可能存在的缓存。
- 重新启动 `npm run dev` 开发服务器。

## 3. 验证与展示
- 自动打开浏览器访问 `http://localhost:3000`。
- 确保页面能正常渲染出（模拟的）博客列表，而不是显示 500 错误页。

这样做的好处是，即使你还没有配置 `.env`，项目也能立即运行起来，你可以先预览 UI 效果，再按照文档从容地去 Notion 获取 API Key。
